CourseFlow Build 1 Markdown



What works now:

* Functional and refined user interface for both the student and faculty views.
* Multiple different routing architecture.
* Tracking of completed evaluations within the system.
* Course evaluation form implementation.



What does not work yet:

* User authentication and authorization, mainly with login validation.
* Inclusion of the official SNHU course catalog.



Build 2 Priorities:

1. Implementing secure user login authorization
2. Deploying traffic control infrastructure (Kubernetes)
3. Integrating of SNHU's course catalog and standardized questionnaire templates into the evaluation system.
